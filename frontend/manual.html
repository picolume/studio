Introduction

PicoLume Studio is a specialized sequencer for choreographing synchronized light shows on low-bandwidth wireless networks. Unlike traditional DMX software that streams raw data, PicoLume sends high-level "commands" (e.g., "Start Rainbow for 5 seconds"). This allows complex visuals across hundreds of props without clogging the RF network.

Interface Overview

The Studio interface is organized to maximize timeline visibility while keeping tools accessible. Hover over buttons for tooltips on their specific functions.

Key Areas

  • Header Toolbar: Project actions (New/Save/Save As/Open), Undo/Redo, playback controls, master volume, and Upload/Export.
  • Effect Palette (Left): A library of all available lighting patterns. These are "Smart Clips" that adapt to any duration you set.
  • Timeline (Center): The main workspace. Top tracks render *above* bottom tracks visually (bottom track = highest priority).
  • Inspector (Right): Context-aware panel. It shows Project Settings when nothing is selected, and Clip Properties when you click a clip. Time values use MM:SS.ss format (e.g., "05:23.45" for 5 minutes, 23.45 seconds). Sections can be collapsed by clicking their headers.
  • Preview Canvas (Top Center): A real-time simulation of your show. Use the preview mode buttons to switch between Track/Field/Off, and drag the resize handle under the preview to change its height.
  • Status Bar (Bottom): Shows project name (with ‚óè when unsaved), Pico device connection status, current selection info, snap/grid settings, and keyboard shortcut hints.

Project Files (.lum)

Studio saves projects as .lum files (ZIP archives containing project.json and embedded audio under audio/).

For safety, Studio enforces limits when opening projects:

  • Max .lum size: 500 MB
  • Max project.json: 10 MB
  • Max single audio file: 200 MB
  • Max total extracted size: 1 GB
  • Max files in archive: 100

If a project exceeds these limits, it will be rejected instead of partially loading.

Note: Text loaded from project files is treated as plain text; HTML markup is not supported and will not be rendered.

Preview Modes

The Preview shows a live approximation of what your props will do at the current playhead time. It is not a pixel-perfect hardware emulator, but it is great for timing and intent.

Track View

Track View displays each LED track as a compact strip, centered in the preview. This is the fastest way to verify layering and timing across tracks.

Field View

Field View displays your props as rings arranged on a field. Each ring shows a sampled slice of the effect so you can quickly see how different props will look at the same moment.

  • Auto layout: Props are arranged automatically in a grid when no saved position exists.
  • Drag to reposition: Click and drag any prop ring to place it where you want. This lets you mirror your real-world stage layout.
  • Positions saved: Custom positions are stored in the project file and restored when you reopen.

Off

Off disables preview rendering. Useful on low-power machines or when you want maximum timeline space.

Resizing the Preview

Drag the horizontal handle directly under the preview canvas to adjust height (80px‚Äì600px range). The app remembers your preview height between sessions.

Layout

You can show/hide major panes to maximize workspace. The layout buttons are in the header (right side) and the app remembers your choices.

  • Toggle Palette: Alt+1
  • Toggle Preview: Alt+2
  • Toggle Inspector: Alt+3

Manual Access

The manual opens inside the app. Press Esc to close it.

Hardware Profiles & Patching

This is the most powerful feature of PicoLume Studio. It allows you to control a "heterogeneous" fleet of props-meaning not all your lights need to be the same length.

Scenario: The Mixed Band

Imagine a marching band show. You have 20 Color Guard Flags (small, 30 LEDs) and 4 Large Field Towers (large, 300 LEDs). You want to run a "Rainbow" effect on everything. With Profiles, you create a "Flag" profile and a "Tower" profile. When you apply the Rainbow effect, it automatically scales: the Flags show a mini rainbow, and the Towers show a massive detailed rainbow.

1. Creating a Profile

Go to the Inspector panel (ensure no clips are selected so you see "Project Settings").

  1. Find the Hardware Profiles section.
  2. Click Add Profile.
  3. Name: Edit the profile name directly in the list (e.g., "Snare Drum").
  4. Props: In the Props field, enter the hardware IDs that should use this profile.
  5. Hardware details: Click the gear to open Hardware Profile Settings, then set:
    • LED Count
    • LED Type (chipset)
    • Color Order (fixes swapped colors)
    • Max Brightness (limits max brightness per prop type)
    • Voltage and Notes (optional documentation)

2. Patching IDs (Props Field)

The Props field tells Studio which physical props belong to this profile. You set the ID on each physical Pico controller using the onboard buttons.

  • Valid range: IDs are in the range 1-224.
  • Syntax: Use comma-separated numbers and ranges.
  • Example: 1-10, 15, 20-25 means props 1 through 10, prop 15, and props 20 through 25 use this profile.

3. Order & Conflicts

Studio generates a patch map automatically from your profile Props assignments when exporting or uploading. If the same prop ID appears in multiple profiles, the lower profile in the list wins. Drag profile cards to reorder if needed, but it's best to avoid overlaps.

Drag to Reorder

Hardware Profiles, Prop Groups, and Color Palettes can all be reordered by dragging their cards up or down within their respective sections.

Color Palettes

Color Palettes are named collections of colors that make it easy to maintain a consistent color scheme across your show. When editing an effect's color property, you can quickly pick from any palette instead of manually entering hex codes.

Built-in Palettes

Studio includes four built-in palettes to get you started:

  • Warm: Reds, oranges, and yellows
  • Cool: Cyans, blues, and violets
  • Ocean: Deep navy to bright cyan gradient
  • Forest: Dark green to lime gradient

Built-in palettes cannot be deleted, but you can duplicate them to create customized versions.

Creating a Custom Palette

In the Color Palettes section of the Inspector (when no clip is selected):

  1. Click + Add Palette.
  2. A new palette opens in the editor with default colors.
  3. Name: Enter a descriptive name (e.g., "Team Colors", "Halloween").
  4. Colors: Click any swatch to change its color, or use the + button to add more colors.
  5. Click Save to confirm your changes.

Editing Palettes

  • Custom palettes: Click the edit icon to modify colors or rename.
  • Built-in palettes: Clicking edit will create a duplicate that you can customize.
  • Delete: Custom palettes can be deleted using the × button. Built-in palettes cannot be deleted.

Using Palettes with Effects

When you select a clip and view its properties in the Inspector, any color property will show:

  1. A Palette dropdown to select which palette to use.
  2. Color swatches from the selected palette for quick selection.
  3. A color picker for custom colors not in any palette.

The palette selection is remembered per-property, so different colors on the same effect can use different palettes.

Tip: Consistent Theming

Create a custom palette with your event's brand colors at the start of a project. This makes it easy to apply consistent colors across all effects without remembering hex codes.

Prop Groups

Prop Groups are logical collections of hardware IDs. While Profiles define *what* the hardware is, Groups define *how you select them* in the timeline.

Scenario: Split Stages

If you want your stage Left props to flash Red, and stage Right props to flash Blue:

  1. Create a Group named "Stage Left" with IDs 1-5.
  2. Create a Group named "Stage Right" with IDs 6-10.
  3. Create two LED Tracks in the timeline.
  4. Assign Track 1 to "Stage Left" and Track 2 to "Stage Right".
  5. Place your effects. They will play simultaneously on their respective hardware.

Creating a Group

In the Prop Groups section of the Inspector:

  1. Click + Add Group.
  2. Name: Enter a name (e.g., "Odd Numbers").
  3. IDs: Enter the IDs (e.g., 1,3,5,7,9).

Timeline Controls

The timeline is the central workspace for arranging your light show. These controls help you navigate and edit precisely.

Playhead & Scrubbing

The playhead (vertical line) shows the current playback position. To move it:

  • Click anywhere on the ruler (top of the timeline) to jump to that time.
  • Drag across the ruler to scrub through the show in real-time.
  • The current time is displayed in the header as MM:SS.ss.

Zoom

Zoom in to make precise edits, or zoom out to see the full show:

  • Ctrl + Scroll (or Cmd + Scroll on Mac) anywhere on the timeline to zoom in/out.
  • Zoom range is 10% to 200%.

Grid & Snap

The grid provides visual timing guides and enables snap-to-grid behavior:

  • Snap checkbox: When enabled, clips align to grid lines when dragging or resizing.
  • Grid size dropdown: Choose from 1s, 1/2s, 1/4s, or 1/8s intervals.
  • Keyboard nudging (arrow keys) also respects the grid size when snap is enabled.

Project Duration

The total project length determines the timeline's scrollable width. To change it:

  1. Click on empty space in the timeline (or press Esc) to deselect all clips.
  2. In the Inspector, find the Duration field under Project Settings.
  3. Enter the desired duration in MM:SS.ss format.

Status Bar

The bar at the bottom of the screen shows: project name (with ‚óè if unsaved), Pico connection status, current selection, snap/grid settings, and keyboard hints.

Working with Tracks

The timeline allows you to layer effects. The system processes tracks from top to bottom, meaning effects on lower tracks effectively "paint over" effects on upper tracks if they target the same props.

LED Tracks

To add lighting, click + LED Track. Each LED track has:

  • Track label: Double-click to rename (e.g., "Verse 1", "Chorus Accent").
  • Target Group dropdown: Select which prop group receives this track's effects.
  • Delete button: Remove the track and all its clips.

Drag effects from the Palette onto an LED track lane to create a clip. You can drag clips left/right to change time, and drag their ends to change duration.

Audio Tracks

Click + Audio Track to add a lane for music. Audio tracks have:

  • Track label: Double-click to rename.
  • Folder icon: Click to browse for an audio file (MP3, WAV, or OGG).
  • Drag & drop: Drag audio files directly from your file explorer onto the track.

Use the visual waveform to align lighting effects with beats and musical cues.

Track rules: Audio clips can only live on Audio tracks. Lighting clips can only live on LED tracks.

Reordering & Moving

  • Reorder tracks: Drag a track header up/down.
  • Move clips between tracks: Drag selected clips onto another track of the same type (LED to LED, or Audio to Audio).

Clip Operations

  • Select: Click a clip. Hold Ctrl/Cmd to add to selection.
  • Move: Drag selected clips, or use arrow keys to nudge by grid size.
  • Resize: Drag clip edges, or use Shift+Arrow keys. Minimum duration is 100ms.
  • Copy/Paste: Ctrl/Cmd+C / V. Pasted clips maintain relative positions.
  • Duplicate: Ctrl/Cmd+D. Creates copies immediately after originals.
  • Delete: Delete or Backspace.

Productivity Tips

Snap: Enable the "Snap" checkbox to align clips to the grid.
Multi-Select: Hold Ctrl/Cmd and click multiple clips to operate on them together.

Cue Points

Cue points are markers you place on the timeline that get embedded into the exported show.bin file. They enable live resync during performances‚Äîif the show drifts out of sync with external music, you can trigger a cue to jump the playback to a known position.

How Cues Work

PicoLume supports 4 cue points labeled A, B, C, and D. Each cue stores a timestamp in the show file. During live playback on the hardware, you can trigger any cue to instantly jump to that position.

Setting Cues

  • Keyboard: Position the playhead where you want the cue, then press Shift+1 through Shift+4 to set cues A‚ÄìD respectively.
  • Timeline: Cue markers appear as labeled flags (A, B, C, D) on the timeline ruler. You can drag them to adjust their position.
  • Inspector: Select a cue marker to view and edit its exact timestamp in the Inspector panel.

Jumping to Cues

Press 1 through 4 to instantly jump the playhead to cue A‚ÄìD. This works in Studio for previewing, and on the hardware during live shows.

Clearing Cues

Select a cue marker and press Delete, or use the Inspector to disable or remove the cue.

Use Case: Live Music Sync

Place cues at major song sections (intro, chorus, bridge, outro). If the lighting drifts from the music during a live performance, the operator can trigger the nearest cue to resync without restarting the entire show.

Upload & Export

Once your show is designed, transfer it to the Master Controller. The file generated includes the event list and the Hardware Profile Lookup Table (LUT).

Option A: Direct Upload (Recommended)

  1. Connect the Pico via USB to your computer.
  2. Hold the CONFIG button on the Pico while plugging it in (or while pressing Reset). The Pico will mount as a USB Mass Storage Device (like a flash drive).
  3. Click the green Upload button in Studio.
  4. The app will automatically find the drive, write the show.bin file, and trigger a device reload.

Reliability on Windows

On Windows, Studio automatically ejects the USB drive after uploading. This ensures all cached writes are flushed before the device reboots, providing maximum reliability. The device also includes retry logic when loading the show file.

Option B: Manual Export

If automatic upload fails, click Export to download the show.bin file. You can then manually drag and drop this file onto the Pico's USB drive.

Binary Inspector

The Binary Inspector is a diagnostic tool for examining and validating show.bin files. It parses the binary format and displays all the data in a human-readable form, helping you verify that exports are correct before deploying to hardware.

Opening the Inspector

  1. Open the hamburger menu (top right).
  2. Click Inspect Binary.
  3. A modal window opens with a drop zone.

Loading a File

  • Drag & drop: Drag a .bin file directly onto the drop zone.
  • Click to browse: Click the drop zone to open a file picker.

Once loaded, the inspector parses the file and displays all available information.

What You'll See

  • Statistics: Event count, total duration, configured props, and file size.
  • Header: Magic bytes (should be "PICO"), version number, and event count from the header.
  • PropConfig: Select a prop from the dropdown to view its LED count, LED type, color order, and brightness cap.
  • Events Table: All events with index, start time, duration, effect name, colors, speed, width, and prop count.

Validation Warnings

The inspector automatically runs validation checks and displays warnings for common issues:

  • Version mismatch: File uses an older format version.
  • Overlapping events: Two events that run at the same time.
  • Zero-duration events: Events with no duration won't display.
  • No props targeted: Events that don't affect any props.
  • Unconfigured props: Events targeting props with 0 LEDs configured.

Filtering Events

Use the "Filter by selected prop" checkbox to show only events that target the currently selected prop. This helps debug issues with specific hardware.

Exporting Data

  • Export JSON: Downloads a structured JSON file with all parsed data (header, propConfigs, events).
  • Export CSV: Downloads a CSV file of the events table for spreadsheet analysis.

When to Use

Use the Binary Inspector when props aren't behaving as expected after upload. Check that: (1) the correct props are targeted by events, (2) PropConfigs match your hardware, and (3) there are no validation warnings.
